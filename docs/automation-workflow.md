# 自動化ワークフロー

## 目的

競合Threadsアカウントの投稿傾向を分析し、差別化された投稿文を作成してThreadsに投稿する。公開投稿は人間の確認を挟む。

## 入力

- 競合アカウントURL
- 自分のアカウントの目的
- 投稿日
- 投稿形式
- 想定読者
- 投稿トーン

指定がない場合は次を標準にする。

- 投稿日：今日
- 投稿形式：ツリー投稿3〜5リプ
- 想定読者：AIで収益化・業務効率化したい個人、クリエイター、事業者
- トーン：わかりやすく、少し先回りした実務目線

## 手順

### 1. 競合アカウント分析

指定された競合アカウントのURLをブラウザで開き、最新の公開投稿を確認する。

分析する項目：

- よく使うテーマ・キーワード（上位3〜5個）
- 投稿の冒頭（フック）の書き方パターン
- ツリー投稿か単発投稿かの比率
- 投稿の頻度感（最新10件の日付から推定）
- エンゲージメントが高そうな投稿の特徴
- 競合がまだ取り上げていない領域（差別化ポイント）

禁止事項：

- 投稿文をそのまま、または少し変えて流用しない
- 自動的な大量取得はしない（手動閲覧の範囲で行う）

### 2. 差別化テーマ選定

競合の傾向を踏まえ、自分のアカウントが出すべきテーマを選ぶ。

選定基準：

- 競合がまだ取り上げていない切り口か
- 自分のアカウントのトーン・読者像と合っているか
- 今日時点で話題性があるか（必要であればWeb検索で補完）
- 読者の得が明確か
- 具体的な行動につなげられるか

### 3. 投稿設計

ツリー投稿は次の構成を標準にする。

- 1投稿目：結論を少し見せて続きを読みたくさせる
- 2投稿目：変化1。何が起きたか
- 3投稿目：変化2。なぜ重要か
- 4投稿目：変化3。どう使えるか
- 5投稿目：注意点。誤解・制約・リスク
- 6投稿目：まとめ。次にやること

リプは3〜5件が標準。

### 4. 投稿前確認

投稿前にユーザーへ提示する。

- 投稿本文
- ツリー構成
- 競合との差別化ポイント
- 注意点
- 投稿するアカウント

ユーザーが明示的に「投稿して」と返したら投稿する。

### 5. Threads投稿

Chromeのログイン済みThreadsを使う。ログイン情報は扱わない。

投稿後はプロフィールまたは投稿ページで公開URLを確認する。

### 6. ログ

templates/post-log.md をコピーして次を記録する。

- 投稿日時
- 投稿URL
- 参考にした競合アカウント
- 差別化したポイント
- 使った出典
- 反応メモ
- 次回改善
