# 自動化ワークフロー

## 目的

AIニュースの調査、SNS向け編集、画像生成、Threads投稿、投稿ログ作成までをCodexで半自動運用する。公開投稿は人間の確認を挟む。

## 入力

- 投稿日
- 対象テーマ
- 投稿形式
- アカウントの目的
- 想定読者
- 投稿トーン

指定がない場合は次を標準にする。

- 投稿日: 今日
- 対象テーマ: AI最新情報
- 投稿形式: ツリー投稿3〜5リプ + 図解投稿1枚
- 想定読者: AIで収益化・業務効率化したい個人、クリエイター、事業者
- トーン: わかりやすく、少し先回りした実務目線

## 手順

### 1. リサーチ

Web検索で当日のAIニュースを調べる。次の順に優先する。

1. 公式発表
2. 企業ブログ
3. 信頼できるニュース媒体
4. 開発者ドキュメント
5. SNS上で話題化している二次情報

確認する項目。

- 発表日
- 発表元
- 何が変わったか
- 誰に影響するか
- すぐ使える示唆
- 誤解されやすい点

### 2. テーマ選定

候補ニュースを3〜5件に絞り、次の観点で評価する。

- 読者の得が明確か
- 投稿当日に話題性があるか
- 具体的な行動につなげられるか
- 図解にしやすいか
- 誤情報リスクが低いか

### 3. 投稿設計

ツリー投稿は次の構成を標準にする。

```text
1投稿目: 今日のAIニュースで重要な変化を予告する
2投稿目: 変化1。何が起きたか
3投稿目: 変化2。なぜ重要か
4投稿目: 変化3。どう使えるか
5投稿目: 注意点。誤解・制約・リスク
6投稿目: まとめ。次にやること
```

リプは3〜5件が標準。情報量が多い場合のみ6件以上にする。

### 4. 図解設計

画像は1080x1080を基本にする。スマホで読めるよう、テキストは少なくする。

おすすめ構成。

- タイトル
- 3つの大きな変化
- それぞれの意味
- 今日からできること
- 小さな注意書き

画像生成プロンプトには次を含める。

- 画角: square, 1080x1080
- 言語: Japanese
- スタイル: clean infographic, modern editorial, high contrast
- 制約: no tiny text, no fake logos, no copyrighted screenshots

### 5. 投稿前確認

投稿前にユーザーへ提示する。

- 投稿本文
- ツリー構成
- 画像
- 出典URL
- 注意点
- 投稿するアカウント

ユーザーが明示的に「投稿して」と返したら投稿する。

### 6. Threads投稿

Chromeのログイン済みThreadsを使う。ログイン情報は扱わない。

投稿後はプロフィールまたは投稿ページで公開URLを確認する。

### 7. ログ

`templates/post-log.md` をコピーして次を記録する。

- 投稿日時
- 投稿URL
- テーマ
- 投稿形式
- 使った出典
- 反応メモ
- 次回改善
