# Threads/SNS Growth Playbook

## 基本方針

Threadsで伸ばす目的は、単発のバズではなく「この人をフォローするとAI活用の理解が速くなる」と覚えてもらうこと。毎回の投稿で、読者に小さな発見か実用的な行動を渡す。

## 伸びやすい投稿の条件

- 冒頭で読み手の関心をつかむ
- 情報が新しい
- 意味づけがある
- 自分ごと化できる
- 保存したくなる
- リプしたくなる余白がある

## フックの型

### 変化予告

```text
今日のAIニュース、地味に見えてかなり重要です。
ポイントは「AIを使う人」より「AIを組み込む会社」が増えること。
続きで3つに分けます。
```

### 誤解訂正

```text
この発表、ただの新機能追加ではありません。
本質は「AIの使い方」が一段変わることです。
```

### 実務目線

```text
AIニュースを見て終わりにするのはもったいないです。
今日の発表から、個人がすぐ真似できる動きだけ拾うとこうなります。
```

### 比較

```text
今日のAIニュースを一言でいうと、
「モデル性能」から「現場導入」へ主戦場が移っています。
```

## ツリー投稿の型

### 1投稿目

- ニュースの全体像
- なぜ今読むべきか
- 続きがあることを自然に示す

### リプ1

- 事実
- 発表元
- 何が起きたか

### リプ2

- 重要性
- 読者への影響
- 市場や働き方の変化

### リプ3

- 具体的な使い方
- 個人・小規模事業者・クリエイターが試せること

### リプ4

- 注意点
- 誤解されやすい点
- まだ様子見すべき点

### リプ5

- まとめ
- 今日やること
- 保存・返信・フォロー導線

## 図解投稿の型

図解は「読み切り資料」ではなく「理解を速くする入口」として作る。

良い図解。

- 1テーマ1枚
- 見出しが強い
- 3〜5要素で整理されている
- 文字が少ない
- 色数が少ない
- スマホで読める

避ける図解。

- 情報を詰め込みすぎる
- 出典が不明
- ロゴやUIスクリーンショットを無断利用する
- 文字が小さい
- 抽象的すぎて何の話かわからない

## 投稿頻度

標準は1日1〜3投稿。

- 朝: ニュース整理
- 昼: 実務に落とした短文
- 夜: 図解またはまとめ

無理に本数を増やすより、同じテーマを複数角度で再利用する。

## エンゲージメント

- 投稿後15〜30分は返信を見る
- 良い質問は次の投稿テーマにする
- 反論や補足は丁寧に拾う
- 反応が良かった表現はテンプレ化する
- 伸びなかった投稿も、フックだけ変えて再投稿候補にする

## AIニュース投稿で大事なこと

- 「すごい」だけで終わらせない
- 読者の仕事や収益にどう関係するかを書く
- 最新情報と実務活用をつなぐ
- 推測は推測と書く
- 発表直後は不確実性もセットで伝える
